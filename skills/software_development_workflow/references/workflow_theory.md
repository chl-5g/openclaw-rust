# 工作流理论

## 什么是工作流

工作流是一系列有序的任务或阶段，每个阶段由特定的 Agent 负责执行。

## 核心概念

### 1. Stage (阶段)
工作流中的独立步骤，如"规划"、"开发"、"测试"等。

### 2. Agent Role (角色)
每个阶段对应的 Agent 角色，如 CEO、PM、Engineer、QA。

### 3. Edge (边)
阶段之间的连接，定义了任务流转的方向。

### 4. Condition (条件)
决定是否触发边转移的条件。

## 工作流模式

### Pipeline (流水线)
线性流程，阶段依次执行：
```
A → B → C → D
```

### Diamond (菱形)
条件分支：
```
    → B →
A →     → D
    → C →
```

### Loop (循环)
带回退的流程：
```
A → B → C → 
    ↑    ↓
    ← ← ←
```

## 状态管理

- **pending**: 等待处理
- **in_progress**: 正在处理
- **blocked**: 被阻塞
- **completed**: 完成
- **failed**: 失败
